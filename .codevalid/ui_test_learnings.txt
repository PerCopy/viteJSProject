2. ExecutionRecorder — CORRECT USAGE
===========================================================================

Constructor signature:  new ExecutionRecorder(acceptanceCriteriaId, testTitle)
  - acceptanceCriteriaId : string  (e.g. "create_event_with_valid_details")
  - testTitle            : string  (e.g. "Create Event Successfully With Valid Inputs")

WRONG — passes Playwright's TestInfo object as second arg:
  const recorder = new ExecutionRecorder("my_id", testInfo);   // TypeError at save()

CORRECT:
  const recorder = new ExecutionRecorder("my_id", "Human-readable test title");

Why it breaks: ExecutionRecorder.save() calls this.testTitle.replace(...).
If testTitle is a TestInfo object (not a string), .replace is undefined → TypeError.
The test assertions may have already passed (CODEVALID_TEST_ASSERTION_OK printed)
before the crash, so the test still fails in Playwright even though the UI worked.

Always call recorder.save(testInfo) at the very end, passing testInfo only there.

3. SELECTOR STRATEGY — WHAT WORKS IN THIS APP
===========================================================================

DO USE:
  page.locator('input[name="fieldName"]')   — reliable; name attrs are always present
  page.getByRole("button", { name: /text/i })
  page.getByRole("heading", { name: "..." })
  page.getByPlaceholder("...")              — safe for text inputs and textareas

DO NOT USE:
  page.getByLabel("Start Date *")   — FAILS in this repo
  page.getByLabel("End Date *")     — FAILS in this repo

Why getByLabel fails here:
  src/components/ui/Input.jsx renders <label> and <input> as siblings inside a
  <div>, with NO htmlFor/id pairing and NO aria-labelledby/aria-label.
  Playwright's getByLabel() requires an ARIA association to resolve the input.
  Without it the locator never matches and the test times out (30 s).

For date inputs specifically use:
  page.locator('input[name="startDate"]')
  page.locator('input[name="endDate"]')

Verify name attributes by reading src/pages/Events.jsx before writing selectors.

4. AUTHENTICATION / PROTECTED ROUTES
===========================================================================

- /events is a protected route (src/components/ProtectedRoute.jsx checks
  localStorage.getItem("token")).
- Call seedAuthenticatedSession(page) BEFORE page.goto("/events").
  It uses page.addInitScript() to inject localStorage entries before any
  navigation, so the ProtectedRoute check passes on first load.

- Keys written to localStorage by seedAuthenticatedSession:
    "token"       — checked by isAuthenticated() in src/utils/api.js
    "user"        — parsed by getCurrentUser()
    "authToken"   — extra alias for safety
    "currentUser" — extra alias for safety

- If a test navigates to /events without seeding auth, the app redirects to
  /signin and all subsequent selectors time out.

5. API MOCKING PATTERNS
===========================================================================

All API calls go through src/utils/api.js using fetch() with no base URL
(proxied by Vite). Intercept with page.route("**/api/...").

Confirmed API paths (read from src/utils/api.js):
  GET  /api/events          — fetch events list
  POST /api/events          — create event; returns created event object
  POST /api/auth/signin     — sign in
  POST /api/auth/signup     — sign up
  GET  /api/registrations/:eventId
  POST /api/registrations

mockEventScenario(page, { initialEvents, createdEvent }) in helpers/mock-api.js:
  - Single page.route("**/api/events") handler covers both GET and POST.
  - Tracks a local `eventCreated` flag; GET returns initialEvents before POST,
    returns [createdEvent] after POST (covers page reload / re-navigation).
  - POST returns HTTP 201 with the createdEvent object.

Always include registrationCount: 0 in event fixture objects — the app
renders event.registrationCount and will show "undefined registered" if missing.

Event object shape expected by Events.jsx:
  { id, title, description, location, startDate, endDate, registrationCount }

7. PLAYWRIGHT CONFIG EXPECTATIONS
===========================================================================

  Use relative paths in page.goto(): page.goto("/events"), not full URLs.

- webServer command: "npm run dev:frontend -- --port 5174 --strictPort"

- Spec files must match **/*.spec.js to be picked up by the project config.

  video: "on", screenshot: "on"). Check test-results/ folder for diagnostics.

  writing a page.route() intercept.
